# SashaOS

Personal operating system project built for learning and experimentation.

## What this project is

SashaOS is my personal playground for understanding how operating systems work under the hood.  
The goal is not to create a production-ready OS, but to explore:

- Low-level concepts (memory, processes, syscalls, interrupts)
- Boot sequence and how a machine starts up
- Basic kernel structures and scheduling ideas
- Simple file and I/O handling
- How higher-level abstractions are built on top of hardware

## What I’m focusing on

- **Learning by doing**: writing small, focused components instead of a big monolith
- **Reading and implementing** concepts from books, articles, and existing OSes
- **Documenting my progress** so others (and future me) can see how I think

## Current status

This is an early-stage, work-in-progress project.  
I am currently working on Windows, using WSL (Ubuntu) as my Linux environment and planning to use QEMU to experiment with bootloaders and my own OS builds.  
Not everything works, and parts will change often as I learn and refactor.
